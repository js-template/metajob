
# Collection Type APIs  
---
## Overview  
Collection type APIs manage lists of content, such as articles, products, or layouts.  


## API Endpoints  

### `GET /api/layouts`  
- **Description**: Fetches a list of layouts from the collection type.  
- **Example Use Case**: Displaying available templates for users to choose from.  

**Playground Link**:  
[Try `GET /api/layouts`](https://your-live-api-link.com/layouts)  

---

### `POST /api/layouts`  
- **Description**: Creates a new layout entry in the collection.  
- **Example Use Case**: Adding a new layout option in the admin panel.  

**Playground Link**:  
[Try `POST /api/layouts`](https://your-live-api-link.com/layouts-post)  

---

### `PUT /api/layouts/:id`  
- **Description**: Updates an existing layout entry by ID.  
- **Example Use Case**: Modifying layout details, like changing the name or sections.  

**Playground Link**:  
[Try `PUT /api/layouts/:id`](https://your-live-api-link.com/layouts-put)  

---

### `DELETE /api/layouts/:id`  
- **Description**: Deletes a layout entry by ID.  
- **Example Use Case**: Removing outdated or unused layouts.  

**Playground Link**:  
[Try `DELETE /api/layouts/:id`](https://your-live-api-link.com/layouts-delete)
